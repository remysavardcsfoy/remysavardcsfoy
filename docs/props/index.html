Props

«Prophes» est court pour les «propriétés».Lorsque React voit un élément représentant un composant défini par l'utilisateur, il transmet des attributs JSX à ce composant en tant qu'objet unique.Nous appelons cet objet «accessoires».

  1. Créez un composant «Greeter» et rendez-le

    function Greeter() {
      return <h1>Hello</h1>;
    }
    ReactDOM.createRoot(document.getElementById('root')).render(<Greeter />);

  2. Ajoutez une «propriété (prop)» au composant

    function Greeter(props) {
      return <h1>Hello, {props.first}</h1>;
    }
    ReactDOM.createRoot(document.getElementById('root')).render(
      <Greeter first="Joe" />
    );

Lecture seulement​

Cette fonction est considérée comme «pure» «pure» car elle n'essaie pas de modifier ses entrées et renvoie toujours le même résultat pour les mêmes entrées.

function sum(a, b) {
  return a + b;
}

En revanche, cette fonction est impure car elle modifie sa propre entrée:

function withdraw(account, amount) {
  account.total -= amount;
}

Tous les composants réagis doivent agir comme des fonctions pures par rapport à leurs accessoires.Les accessoires sont immuables, ils ne peuvent pas être modifiés.

  1. Mettez à jour le composant greeter pour modifier ses« accessoires »

    function Greeter(props) {
    +  props.first = 'Dave';
      return <h1>Hello, {props.first}</h1>;
    }
    ReactDOM.createRoot(document.getElementById("root")).render(
      <Greeter first="Joe" />
    );

  2. Ouvrez Devtools de votre navigateur et vous recevrez l'erreur suivante:

    Uncaught TypeError: Cannot assign to read only property 'first' of object '#<Object>'
     at Greeter (<anonymous>:4:14)

  3. Retirez la modification des «accessoires»

    function Greeter(props) {
    -  props.first = 'Dave';
      return <h1>Hello, {props.first}</h1>;
    }
    ReactDOM.createRoot(document.getElementById("root")).render(
      <Greeter first="Joe" />
    );

  4. L'erreur disparaîtra et le composant rendra à nouveau.

Bien sûr, les UIS des applications sont dynamiques et changent avec le temps.Dans la section suivante, nous présenterons un nouveau concept de «État».L'état permet aux composants React de modifier leur sortie au fil du temps en réponse aux actions des utilisateurs, aux réponses du réseau et à toute autre chose, sans violer cette règle. accessoires destructureurs​

Les accessoires destructuration sont courants et considérés généniens comme la meilleure pratique pour réagir car il rend le code plus facile à lire et à comprendre. Il est plus utile car plus de propriétés sont ajoutées Les exemples ci-dessous évoluent pour démontrer les avantages de la syntaxe lorsqu'ils traitent des «accessoires».

function Greeter(props) {
  return (
    <h1>
      Hello, {props.first} {props.last}
    </h1>
  );
}
ReactDOM.createRoot(document.getElementById('root')).render(
  <Greeter first="Srini" last="Kata" />
);
function Greeter(props) {
  // const first = props.first;
  // const last = props.last;
  const { first, last } = props;
  return (
    <h1>
      Hello, {first} {last}
    </h1>
  );
}
ReactDOM.createRoot(document.getElementById('root')).render(
  <Greeter first="Srini" last="Kata" />
);
function Greeter({ first, last }) {
  // const { first, last } = props;
  return (
    <h1>
      Hello, {first} {last}
    </h1>
  );
}
ReactDOM.createRoot(document.getElementById('root')).render(
  <Greeter first="Srini" last="Kata" />
);

1

const element = <Greeter name="Joe" />;
  1. Si vous souhaitez attribuer des propriétés à une variable ou à une propriété d'objet, vous devez utiliser une expression et laisser les devis.
    const person = { firstName: 'John' };
    const element = <Greeter name={person.firstName} />;
  2. Renders: Hello John
  3. Si vous quittez les citations, la réact vous prend littéralement.
    const person = { firstName: 'John' };
    const element = <Greeter name="{person.firstName}" />;
  4. Renders:
    Hello, {person.firstName}

Fonction par rapport aux composants de classe​

Étant donné que les méthodes de classe créent un nouveau contexte de fonction (ceci), assurez-vous d'utiliser «this.props» dans un «composant de classe» au lieu de «accessoires» comme vous le feriez dans un «composant de fonction» pour référence aux propriétés transmises dans le composant.

  • Function Component: props
  • Class Component: this.props
  1. Mettez à jour votre composant de fonction pour être un composant de classe et assurez-vous d'utiliser this.props
    class Greeter extends React.Component {
      render() {
        return <h1>Hello, {this.props.name}</h1>;
      }
    }
