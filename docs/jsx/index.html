JSX

JSX fournit juste du sucre syntaxique pour la fonction :

React.createElement(component, props, ...children);

Partie 1 - Remplacer createElement par JSX​

  1. Commentez l'appel React.createElement et remplacez-le par JavaScript XML (JSX) comme indiqué ci-dessous. Une fois compilé, le JSX générera le code commenté en dessous.

    //    const element = React.createElement(
    //      'div',
    //      {
    //        className: 'container'
    //      },
    //      'Hello World'
    //    );

    const element = <div className="container">Hello World</div>;
    ReactDOM.createRoot(rootElement).render(element);

  2. Mais comme un compilateur JSX n'a pas été configuré, si vous essayez d'exécuter ce code en ouvrant ou en actualisant la page d'index dans votre navigateur, vous obtiendrez l'erreur suivante :

    Uncaught SyntaxError: Unexpected token <

Parce que la syntaxe des parenthèses (JSX) n'a pas été compilée en JavaScript.

  1. Pour installer le compilateur JSX (Babel)

    npm install --save-dev @babel/standalone

Compiler dans le navigateur avec @babel/standalone a un cas d'utilisation assez limité, donc si vous travaillez sur un site de production, vous devriez précompiler vos scripts côté serveur.

  1. Configurer le compilateur JSX (Babel). Effectuez les modifications suivantes :

    ...
      <script src="/node_modules/react-dom/umd/react-dom.development.js"></script>
    + <script src="/node_modules/@babel/standalone/babel.min.js"></script>
    - <script type="text/javascript" src="/main.js">
    + <script type="text/babel" src="/main.js">
    ...
    </script>

  2. Vous devriez maintenant voir à nouveau la sortie suivante sur la page

    Hello World

  3. Dans le chapitre précédent, nous avons écrit les appels React.createElement mais maintenant le compilateur Babel les génère.

    Si vous voulez tester comment certains JSX spécifiques sont convertis en JavaScript, vous essayez [le compilateur Babel en ligne] (https://babeljs.io/repl)

    Voici quelques autres exemples :

    Liste:

    <ul>
      <li>Home</li>
      <li>About</li>
      <li>Contact</li>
    </ul>
    React.createElement(
      'ul',
      null,
      React.createElement('li', null, 'Home'),
      React.createElement('li', null, 'About'),
      React.createElement('li', null, 'Contact')
    );

Collez [ce formulaire de connexion] (./snippets/bootstrap-form.jsx) dans [le compilateur Babel en ligne] (https://babeljs.io/repl) pour comprendre pourquoi vous souhaitez utiliser JSX comme votre HTML grandit.

Partie 2 - Intégration d'expressions dans JSX​

Une expression peut être créée dans JSX avec des accolades. Lorsque vous créez une expression à l'aide d'accolades, vous dites que vous voulez écrire du JavaScript.

  1. Créez une variable et attribuez-lui un "nom", puis utilisez cette variable dans votre "JSX" dans une "expression".

    <script type="text/babel">
      const rootElement = document.getElementById('root');
    +  const first = 'Joe';
    +  const element = <div className="container">Hello {first}</div>;
     ReactDOM.createRoot(rootElement).render(element);
    </script>
    Hello Joe

    Vous pouvez également utiliser des objets complexes :

     const rootElement = document.getElementById('root');
    +  const person = { first: 'Joe', last: 'Doe' };
      const element = (
        <div className="container">
    +      Hello {person.first} {person.last}
        </div>
      );
     ReactDOM.createRoot(rootElement).render(element);

Partie 3 - Spécification d'attributs avec JSX​

  1. Téléchargez le logo React depuis ce site à la racine de votre projet demos en tant que react-logo.png.

  2. Mettez à jour l'élément pour qu'il devienne une balise d'image pointant vers le logo.

    const element = <img src="./react-logo.png" />;

  3. Actualisez la page et le logo React devrait apparaître sur la page.

  4. Créez un objet logo avec des propriétés de chemin, de nom et de titre et définissez des attributs dans JSX sur chacune de ces valeurs, comme indiqué ci-dessous.

  5. const logo = {
      name: 'React Logo',
      title: 'React Logo',
      path: './react-logo.png',
    };

    const element = <img src={logo.path} alt={logo.name} title={logo.title} />;

  6. Actualisez la page et le logo React devrait toujours apparaître sur la page. Survolez le logo avec votre souris et le titre devrait apparaître.

    Notez que lorsque la valeur est un "littéral de chaîne", vous pouvez simplement définir l'attribut dessus, mais lorsque la valeur fait référence à une variable ou à une propriété sur un objet, vous devez utiliser la syntaxe "expression" et assurez-vous de laisser de côté les guillemets '''`.

  7. En général, vous pouvez utiliser n'importe quelles propriétés et attributs DOM dans JSX, mais il y a quelques points à prendre en compte :

    Dans React, toutes les propriétés et tous les attributs DOM (y compris les gestionnaires d'événements) doivent être en camelCased. Par exemple, l'attribut HTML tabindex correspond à l'attribut tabIndex dans React. L'exception concerne les attributs aria- et data-, qui doivent être en minuscules. Par exemple, vous pouvez conserver aria-label comme aria-label.

    Il existe un certain nombre d'attributs qui fonctionnent différemment entre React et HTML. Vous pouvez en savoir plus ici.

Les différences les plus courantes sont présentées ci-dessous.

  • Au lieu de class, utilisez className

    <div className="alert alert-danger">error message</div>
  1. Remplacez la variable d'élément par la div indiquée ci-dessus avec un message d'erreur 2. Actualisez la page dans votre navigateur et inspectez la div pour voir que className a été remplacé par class

  • Au lieu de for utiliser htmlFor

    <form>
      <label htmlFor="name">Name</label>
      <input id="name" />
    </form>

Partie 4 - Enfants​

  1. Vous pouvez imbriquer plusieurs éléments enfants.

    const element = (
      <ul>
        <li>
          <a href="">a</a>
        </li>
        <li>
          <a href="">b</a>
        </li>
        <li>
          <a href="">c</a>
        </li>
      </ul>
    );

  2. Si vous voulez que la mise en forme et l'indentation de votre code soient alignées, vous voudrez que votre balise d'ouverture soit sur la ligne suivante après la déclaration de la variable. Étant donné que JavaScript a une insertion automatique de points-virgules, votre code peut se casser comme indiqué ci-dessous.

    const element = ;
     <ul>
     ...

  3. Pour éviter les problèmes d'insertion automatique de points-virgules et toujours pouvoir utiliser l'indentation appropriée, placez simplement votre JSX entre parenthèses () comme indiqué ci-dessus.

  4. Bien que vous puissiez imbriquer n'importe quel nombre d'éléments dans l'élément parent racine, vous ne pouvez avoir qu'un seul élément parent. Le code ci-dessous est invalided.

    const element = (
        <p>123</p>
        <ul>
        . ..
        </ul>
    );

    Vous recevrez l'erreur suivante :

    Uncaught SyntaxError: /Inline Babel script: Adjacent JSX elements must be wrapped in an enclosing tag. Did you want a JSX fragment <>...</>?

Pour résoudre ce problème, vous pouvez soit :

- enveloppez votre code dans un élément conteneur comme un div
const element = (
+     <div> //extraneous element
      <p>123</p>
      <ul>
      . ..
      </ul>
+    </div>
  );

  • enveloppez votre code dans un élément React.Fragment

    const element = (
    +     <React.Fragment>
          <p>123</p>
          <ul>
          . ..
          </ul>
    +    </React.Fragment>
      );

  • utiliser une syntaxe abrégée

    La prise en charge de la syntaxe des fragments dans JSX varie en fonction des outils que vous utilisez pour créer votre application.

    const element = (
    +     <>
          <p>123</p>
          <ul>
          . ..
          </ul>
    +     </>
      );
